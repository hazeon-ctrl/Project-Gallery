# OctoAlien - Design Specification v2.1) October 4, 2025

**Format:** Game Program Design Specification
**Intended Audience:** Claude.ai and other implementation agents
**Target Platform:** Mobile-first (touch), PC-compatible (keyboard)
**Unique formatting: "+" focus on this element +, "~" add this element if possible ~

---

## Core Design Pillars

1. **Retro Arcade Authenticity** - Evoke 1970s-1980s arcade cabinets through visual style, sound design, and game feel
2. **Mobile-Native Experience** - Touch controls as primary input, layout optimized for vertical mobile displays
3. **Rhythmic Gameplay** - All timing, movement, and audio should feel connected and musical ~
4. **Signature Mechanic Focus** - The three-stage Charge Bomb is the star; make it feel powerful and satisfying +

---

## Visual Design System

### Aesthetic Guidelines

- **Style:** Retro arcade with deliberate detail + - **Philosophy:** Simple but not basic - small touches matter (eyes on bricks, tentacle animation, subtle gradients)
- **Effects:** CRT-style glow/bloom, slight color bleed on bright elements, scanline overlay optional +
- **Typography:** Monospace font for basic text and numbers, additional messages or displays may be contrasting ~
- **Color Temperature range:** Warm (magenta, orange, yellow) versus cool (cyan, blue) for visual contrast ~

### Color Palette

- **Primary (Octo):** Magenta, purple, or hot orange - should feel alien and threatening
- **Secondary (Bricks):** Orange spectrum (amber to burnt orange)
- **Tertiary (Bombers):** Brown to orange-brown, darker accents
- **Player:** Light cyan/blue body, light yellow core detail, thin black outline

- **Projectiles:**
- Player bullets: Bright yellow with glow
- Brick bullets: Orange with glow
- Bomber bullets: Dark red or brown, minimal glow
- Octo bombs: Match Octo color with lighter border/glow
- **Background:** Deep space - black base with white stars, subtle blue nebula hints
- **UI Text:** White or cyan, always readable against background

### Visual Hierarchy & Layout
**Mobile Vertical Display Priority**

```
┌─────────────────────┐
│ OCTO ALIEN (5%) │ ← Top anchor
├─────────────────────┤
│ ▓▓▓▓ BRICK ROWS │ ← 4 rows, spawning zone
│ ▓▓▓▓ (15-20%) │
│ ▓▓▓▓ │
│ ▓▓▓▓ │
├─────────────────────┤
│ │
│ [BOMBER ZONE] │ ← Mid-screen traverse
│ │
│ SCORE: 0000 ← │ ← Stats overlay (no border)
│ LIVES: ❤❤❤ │ Middle-left, medium size
│ HEALTH: ████ │
│ │
├─────────────────────┤
│ △ │ ← Player zone (bottom 15%)
│ [CHARGE] │ ← Charge button bottom-left
└─────────────────────┘
```

**Element Sizing:**
- Octo: ~15-20% screen width, visible tentacles extend below body
- Bricks: Small, 10-15 per row with small gaps
- Player: ~6-8% screen width, clearly visible
- Charge Button: ~8-10% screen width, similar to player size, easy to tap
- UI Text: 3-4% screen height, crisp and legible

## Game Objects - Detailed Specifications

### The Octo Alien (Boss)
**Visual Description:**
- Base shape: Colored rectangle (landscape orientation)
- Two horizontally rectangular eyes: white, positioned near top of base shape
- Eight tentacles along bottom edge: flowing, organic movement +
- Color: Magenta/purple/hot-orange gradient or solid
- Subtle glow effect around entire body
- Size: Should feel imposing but leave room for brick rows below +

**Tentacle Animation:**
- Formula-based sine wave motion: `offset = sin(time * speed + tentacleIndex * phase) * amplitude`
- Suggested values: speed = 2-3, phase = 0.4-0.5 radians, amplitude = 8-15 pixels
- Creates gentle, hypnotic undulation ~
- Should feel alien but not chaotic

**Behavior:**
- **Phase 1 (100%-50% health):** Stationary at top center, menacing but still
- **Phase 2 (49%-0% health):** Horizontal movement, sine wave or back-forth pattern, drops bombs every 2-3 seconds
- Movement speed: Moderate, telegraphed, not frantic

### Bricks (Secondary Enemies)
**Visual Description:**
- Small rectangles, orange spectrum (amber, orange, burnt orange variations)
- Optional detail: Two tiny white dot eyes for personality
- Slight gradient or highlight for depth
- Minimal glow

**Behavior:**
- Spawn randomly in available slots across 4 rows
- Spawn rate: Start slow (~1 brick per second), increase gradually
- Each brick fires bullets downward at intervals (0.5-1.5 seconds randomly)
- Destroyed on contact with player bullet (1 hit)
- Score value: 10 points per brick

**Bullet Pattern:**
- Straight down trajectory
- Orange glow trail
- Moderate speed (dodgeable but not trivial)

### Bombers (Tertiary Enemies)
**Visual Description:**
- Wider rectangle body (2:1 or 3:1 aspect ratio)
- Smaller rectangle markers on left and right edges (darker shade)
- Brown to orange-brown color
- Minimal detail, functional design

**Behavior:**
- Spawn randomly from left or right edge
- Move horizontally across screen following gentle sine wave (vertical oscillation)
- Drop 1-3 bombs during crossing
- Spawn frequency: Every 8-15 seconds (variable)
- Speed: Moderate, trackable
- Can be destroyed by player bullets (2-3 hits)
- Score value: 50 points per bomber

### The Player
**Visual Description:**
- "Baseball diamond" rotated: right angle point at top, 90° arc at bottom
- Light blue/cyan base color
- Light yellow central feature (cockpit/core)
- Thin black outline for definition
- Slight glow or pulse when charging bomb

**Controls:**
- **Touch (Mobile):** Tap/drag left or right side of screen to move player
- **Keyboard (PC):** A/D or Left/Right arrows
- Movement: Smooth, responsive, slight momentum but snappy
- Auto-fire while moving: 3 shots per second, evenly spaced

**Stats:**
- Health: 100% (displayed as bar or percentage)
- Lives: 3 (displayed as hearts or icons)
- Takes 25% damage from Octo bombs
- Loses 1 life from brick or bomber bullets
- Death: Respawn after 1 second if lives remain

### Charge Bomb System (Signature Mechanic)

**UI Button:**
- Position: Bottom-left corner
- Size: Similar to player character (~8-10% width)
- Label: "CHARGE" in monospace font
- Visual states:
- Idle: Muted color (gray or dark blue)
- Charging: Pulsing glow, intensity increases with charge time
- Launched: Brief flash, returns to idle
- Cooldown: Dim with timer or fill indicator

**Three-Stage Mechanic:**

**Stage 1 - Charge (Press and Hold):**
- Duration: 0-2 seconds (capped at 2s)
- Visual feedback:
- Button glows brighter over time
- Player character pulses/glows with energy
- Optional: Charge meter fills, screen edges glow
- Audio: Rising pitch or intensifying hum (110Hz → 440Hz over 2 seconds)
- Feel: Building tension and power

**Stage 2 - Launch (Release):**
- Bomb launches upward from player position
- Visual: Bright yellow-white sphere with glow trail
- Speed: Fast but trackable (~400-600px/second)
- Audio: Sharp "whoosh" release sound (spike at 880Hz)
- Bomb travels until player detonates or it reaches top of screen

**Stage 3 - Detonate (Press Again):**
- Player presses Charge button again to trigger detonation
- Visual effect:
- Expanding circular rectangular wave from bomb position
- Bright flash (white/yellow)
- Blast radius visualization (circle or hexagon, semi-transparent)
- Radius based on charge time: min ~60px, max ~200px
- Audio: Deep "BOOM" with satisfying bass (80-110Hz) and bright overtones
- Duration: Blast visual persists for 0.5-1 second
- Effect: Prevents brick spawning in blast radius for 3-5 seconds
- Cooldown: 4-6 seconds before next charge available

**Blast Area Mechanic:**
- Creates temporary "dead zone" where bricks cannot spawn
- Does NOT destroy existing bricks (strategic choice)
- Does NOT affect Octo or Bombers
- Visual indicator: Faint shimmer or grid distortion in affected area during prevention period

## Audio Design System

**Philosophy:** Every action should have a distinct, musically related sound using Web Audio API

### Tonal Framework
- **Base Frequency:** A220 (220Hz) - low A
- **Scale:** Pentatonic minor for cohesion (A, C, D, E, G)
- **Span:** 2-3 octaves (110Hz - 880Hz)
- **Synthesis:** Simple waveforms (sine, triangle, square, sawtooth)

### Sound Map

**Player Actions:**
- Move: Subtle low pulse (110Hz triangle wave, 50ms)
- Shoot: Quick ping (440Hz sine, 80ms, slight pitch bend down)
- Hit Enemy: Satisfying "pop" (330Hz → 220Hz, 100ms)
- Take Damage: Harsh drop (440Hz → 110Hz sawtooth, 200ms)
- Death: Descending arpeggio (440→330→220→110Hz, 500ms total)

**Enemies:**
- Brick Spawn: Soft "bleep" (330Hz sine, 60ms)
- Brick Shoot: Sharp "pew" (550Hz triangle, 70ms)
- Bomber Approach: Low rumble (110Hz square with LFO, continuous)
- Bomber Bomb Drop: Deep "thunk" (165Hz, 120ms)

**Octo Alien:**
- Idle Ambience: Slow LFO on 220Hz (ominous presence)
- Phase Change (50% health): Dramatic tone shift (chord: 220Hz + 277Hz + 330Hz)
- Movement: Swelling pulse synchronized with position
- Drop Bomb: Threatening drop (550Hz → 165Hz, 200ms)
- Death: Complex descending cascade (500ms, multiple frequencies)

**Charge Bomb:**
- Charging: Rising pitch (110Hz → 440Hz over charge time, sine wave)
- Launch: Sharp "fwoosh" (880Hz spike, quick decay)
- Detonate: Layered explosion (80Hz bass + 440Hz treble, 300ms)
- Blast Active: Subtle shimmer (high frequency noise, quiet)

**UI/System:**
- Game Start: Ascending arpeggio (energizing)
- Level Complete: Victory fanfare (major scale sequence)
- Game Over: Melancholy descent

**Mix Guidelines:**
- Player sounds: Slightly louder, centered - Enemy sounds: Balanced, slight stereo positioning based on screen position
- Bass frequencies (below 150Hz): Use sparingly for impact
- All sounds: Quick attack, short sustain, minimal release (arcade-style)

## Gameplay Flow & Progression

### Game Initialization
1. Title screen with "OCTO ALIEN" in glowing text
2. Brief control overlay (3-5 seconds): "TAP TO MOVE • CHARGE TO BLAST"
3. Fade to game field with stats visible

### Phase 1: Learning Phase (100%-50% Octo Health)
**Tempo: Moderate, Manageable**
- Octo remains stationary
- Brick spawn rate: 1 per 1.0-1.5 seconds
- Bombers: Every 12-15 seconds
- Player learns controls and charge bomb timing
- Feel: Tense but fair

### Phase 2: Challenge Phase (49%-0% Octo Health)
**Tempo: Intense, Rhythmic**
- Octo begins horizontal movement
- Octo drops bombs every 2-3 seconds
- Brick spawn rate: 1 per 0.5-0.8 seconds
- Bombers: Every 8-10 seconds
- Player must juggle multiple threats
- Feel: Chaotic but conquerable with skill

### Victory Condition
- Octo health reaches 0%
- Dramatic death animation (explosion, tentacles go limp, fade out)
- Victory fanfare
- Final score display
- Option to restart

### Defeat Condition
- Player loses all lives
- Brief death animation
- "GAME OVER" text
- Final score display
- Option to restart

### Scoring System
- Brick destroyed: 10 points
- Bomber destroyed: 50 points
- Octo damaged: 5 points per 1% health
- Octo defeated: 500 bonus points
- Perfect charge bomb (2s charge): 25 bonus points

## Game Feel & Polish

### Movement Feel
- **Player:** Responsive with slight acceleration/deceleration (50-100ms ease)
- **Octo:** Smooth, menacing, telegraphed
- **Bombers:** Flowing, predictable sine wave
- **All Projectiles:** Crisp, no lag, clear trajectories

### Impact & Feedback
- **Hit Confirmation:** Screen shake (subtle, 2-3px, 50ms) + sound + brief flash on target
- **Charge Bomb Detonation:** Moderate screen shake (5-8px, 200ms) + bright flash
- **Octo Phase Change:** Screen flash + dramatic sound + brief slow-motion (optional)
- **Player Death:** Quick flash to white, 500ms respawn delay

### Particle Effects (Optional but Recommended)
- Bullet impacts: Small burst (3-5 particles)
- Explosions: Expanding ring + particles
- Charge bomb trail: Glowing wake
- Destroyed enemies: Brief fragment scatter

### Performance Targets
- 60 FPS on modern mobile devices
- Touch response latency: <50ms
- Smooth animations, no stuttering
- Efficient collision detection

## Technical Implementation Notes

### Web Audio API Sound Generation
- Use OscillatorNode for tones
- Use GainNode for volume envelopes
- BiquadFilterNode for texture (optional)
- Pre-generate sound patterns for consistency
- Keep total simultaneous sounds <10 for performance

### Collision Detection
- Simple bounding box for most objects
- Circle collision for charge bomb blast radius
- Spatial partitioning for brick grid optimization

### State Management
- Game states: MENU, PLAYING, PAUSED, GAME_OVER, VICTORY
- Octo phases: PHASE_1 (stationary), PHASE_2 (moving)
- Charge bomb states: IDLE, CHARGING, LAUNCHED, DETONATING, COOLDOWN

### Mobile Optimization
- Touch target minimum 44x44px (accessibility)
- Test on various screen sizes (320px - 428px width common)
- Prevent scroll/zoom on game canvas
- Handle orientation lock (portrait only)

` **Design Philosophy Summary:**
OctoAlien should feel like discovering a lost arcade cabinet from 1982 - simple to understand, difficult to master, with a unique central mechanic (charge bomb) that rewards timing and strategy. Every visual and audio element should reinforce the retro aesthetic while providing clear, satisfying feedback. The mobile-first approach means large touch targets, clear visual hierarchy, and gameplay that works in portrait orientation with one-handed controls when possible.

**Design Mantra:** *Retro aesthetics, modern feel. Simple to learn, satisfying to master.*
